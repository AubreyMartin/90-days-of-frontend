# Project Types

The trainer assigned **3 front-end design projects**:

- **Landing page**
- **Business website**
- **E-commerce web application**

The work is **design-focused only for now** — using **HTML, CSS, Bootstrap**, while **JavaScript and jQuery** will be added later.

# Key Requirements for All Projects

- Build **original projects**, not direct clones
- Choose **different concepts** from each other
- Make the UI **clean, polished, and realistic**
- Use **proper colors, fonts, images, and content**
- Treat them like **real client projects**
- The projects should help you explain your work in interviews
- ChatGPT can be used for content ideas, but the content must still be relevant and meaningful

# 1) Landing Page

This is a **single-product / single-concept website**.

## What it should include

- **Minimum 10 sections**
- Can be for a product like a shoe, water bottle, smart device, etc.
- Example structure mentioned:
  - Header
  - Hero section
  - About section
  - Features
  - Testimonials
  - CTA
  - Product highlights
  - Reviews
  - FAQ or similar

## Important notes

- It should feel like a **product launch page**
- Only **one main product or concept**
- Design quality matters a lot

# 2) Business Website

This is for a **service-based business or company**.

## What it should include

- **Minimum 10 pages**
- Focus on a company that provides **multiple services**
- Examples given:
  - Institute
  - Hospital
  - Restaurant group
  - Software company
  - Photographer
  - Mehandi artist
  - Hotel
  - Any service business

## Key idea

- This is **not mainly about selling products**
- It should explain:
  - What the business does
  - Services offered
  - Branches / team / about / contact
  - Client-facing information

## Examples of pages

- Home
- About
- Services
- Individual service pages
- Team
- Testimonials
- Gallery
- Contact
- Branches
- FAQ

# 3) E-commerce Web Application

This is the **main project** and the most important one.

## What it should include

An **end-to-end shopping flow** with multiple screens.

## Pages / screens mentioned

- Homepage
- Category page
- Product listing page
- Product details page
- Cart page
- Checkout page
- Success / order confirmation page
- My orders page
- Login page if needed

## Flow expectation

- User browses products
- Opens product details
- Adds to cart
- Proceeds to checkout
- Sees confirmation / success page
- Can return to home and orders

## Important notes

- Design only for now, logic later
- Every button and screen should be properly designed
- This should feel like a **real e-commerce experience**
- It is the one to keep in the **resume**

# Project Strategy Recommended

- Start by **choosing unique concepts** for each project
- Research 3–4 reference websites for ideas
- Build the structure first
- Add content, sections, and visual polish
- Make sure each project is **different** from others in the batch
- Be ready to explain the project clearly in interviews
